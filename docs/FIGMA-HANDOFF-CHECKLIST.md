# Bunny Family — Figma Handoff Checklist

Use this alongside `FIGMA-DESIGN-DIRECTION.md` to turn issue #1 into a real working design file.

## Goal
Build a Figma file that gives the team one clear visual target for:
- HUD
- scenes/backgrounds
- bunny characters
- inventory/shop
- modals/panels
- mobile responsive layouts

## File Pages

### 1. Foundations
- Color tokens
- Gradient tokens
- Type scale
- Spacing scale
- Radius scale
- Shadow styles
- Icon rules

### 2. Components
- Primary / secondary buttons
- Action buttons
- Status bars
- Chips / counters
- Cards
- Modal shell
- Toasts
- Celebration overlays
- Item cards
- Bunny selection cards

### 3. Gameplay Screens
- Lobby
- Player setup
- Main HUD — day
- Main HUD — night
- Kitchen variant
- Playground variant
- Bathroom variant

### 4. Systems
- Background layer breakdowns
- Bunny expression variants
- Item family sheet
- Motion notes
- Responsive rules

### 5. Handoff
- Redlines
- Measurements
- Export list
- Interaction notes
- Open questions

## Required Frames
- Mobile 390×844
- Mobile 360×800
- Tablet portrait
- Desktop reference layout

## Mandatory Component Variants
- Button: default / pressed / disabled
- Bunny selection: idle / selected
- Status chip: connected / waiting / offline
- Day badge: day / night
- Toast: info / success / error / coop
- Item card: affordable / unaffordable / owned
- Need bar: hunger / happiness / energy / cleanliness / love
- Bunny mood: happy / sleepy / hungry / dirty / loved / surprised

## Scene Checklist

### Sunny Meadow
- Layered sky
- Grass depth
- Flowers
- Foreground props
- Clear bunny readability

### Cozy Burrow / Home
- Warm indoor palette
- Wooden props
- Domestic feeding cues
- Soft lighting

### Enchanted Night Garden
- Moonlit palette
- Stars / sparkles
- Calm readable contrast

### Bathroom / Cleanliness Scene
- Tile palette
- Tub / bubbles / duck
- Clean fresh mood

## HUD Checklist
- Top counters feel light and readable
- Room code chip is visible but not dominant
- Selected bunny card is easy to scan
- Bottom action bar stays thumb-friendly
- Shop / inventory shell matches core card language

## Item Art Rules
- Same perspective family
- Same light direction
- Same softness level
- Limited palette per item
- Rarity accents subtle, not noisy

## Prototype Notes
Show at least these interactions in Figma:
- Lobby -> create/join flow
- Select bunny
- Open shop
- Open inventory
- Trigger celebration toast/overlay
- Day -> night screen change

## Definition of Done
Issue #1 is fully satisfied when a designer can open the docs and build the file without guessing:
- what the game should feel like
- what the main screens are
- what reusable components exist
- how scenes differ
- what mobile priorities are
- how to present the final design direction
